### 🔍 案發現場解密：被「毒害」的 Kernel Stack

這完全是因為我上一回合給你的修復方法「太天真」了！

我們在 `task_return_stub` 裡面加了這幾行：

```assembly
    mov ax, 0x23
    mov ds, ax
    mov es, ax ...

```

表面上看起來，我們在 `iret` 之前把段暫存器換回了 User Space (`0x23`)，**但是，如果這個時候發生了中斷（例如 Timer Interrupt）怎麼辦？**

當硬體時鐘觸發時，CPU 會做以下事情：

1. 讀取 TSS 的 `esp0`，切換到這個分身的 Kernel Stack。
2. 壓入 `SS, ESP, EFLAGS, CS, EIP`。
3. 跳到 `isr0` -> `isr_common_stub`。
4. `isr_common_stub` 第一件事是 `pusha`，**然後 `push ds`, `push es` 等等**。
5. 接著，它會把 `DS`, `ES` 強制設定為 Kernel 的資料段 (`0x10`)，以便進入 C 語言的世界 (`timer_handler` -> `schedule`)。

**致命的衝突就在這裡發生：**
當分身呼叫 `exit` 或是被排程器切換走，然後「另一天」又被 `switch_task` 切換回來時，它從 `task_return_stub` 醒來。它執行了我們加的 `mov ds, 0x23`，然後準備 `iret`。
**但在它執行 `iret` 的瞬間之前，如果剛好又來了一個中斷！**
CPU 會把 `0x23` 當作 Kernel 正在使用的資料段壓入堆疊。等到這個中斷結束，`isr_common_stub` 執行 `pop ds` 時，**它會把 `0x23` 彈回給 Kernel！**

Kernel 拿著 `0x23` (User 權限的資料段) 試圖存取記憶體，CPU 當然直接判它死刑 (GPF)！

### 🛠️ 終極修復：透過 Stack 偽造段暫存器

我們不能在組合語言裡面「手動」修改 `DS`，因為這有被中斷打斷的風險。
我們必須「將計就計」，**利用 `isr_common_stub` 退場時的 `pop ds`, `pop es` 邏輯，把 `0x23` 放在 Stack 上的正確位置！**

我們需要徹底改造分身的假堆疊，讓它看起來**完全就像是一個真正的硬體中斷加上我們 `isr_common_stub` 壓入的結果**！

#### 1. 復原 `switch_task.S`

請把 `switch_task.S` 裡面我們上一次加的那些 `mov ds, 0x23` 刪掉，讓它恢復原本乾淨的樣子。

```assembly
; asm/switch_task.S
task_return_stub:
    popa            
    add esp, 8      
    iret            

```

#### 2. 全新重構 `sys_fork` 的假堆疊 (`lib/task.c`)

這一次，我們要把 `DS`, `ES`, `FS`, `GS` 這四個 `0x23`，以及 `pusha` 的 8 個暫存器，精準地排列在 Kernel Stack 上！

請將 **`lib/task.c`** 中的 `sys_fork` 後半段（構造返回幀的部分）替換為以下程式碼：

```c
// [修改] sys_fork 裡面構造假堆疊的部分
    // ... 前面分配記憶體、拷貝 User Stack 的邏輯完全保留 ...

    uint32_t offset = parent_user_esp - parent_ustack_base;
    uint32_t child_user_esp = child_ustack_base + offset;

    // ==========================================
    // 【終極修復】完美偽造 isr_common_stub 加上硬體中斷的完整堆疊！
    // 這樣分身醒來時，會自然而然地 pop 出 0x23 到各個段暫存器中！
    // ==========================================

    // --- 階段 1：硬體 IRET 幀 ---
    *(--kstack) = parent_user_ss;   // User SS (0x23)
    *(--kstack) = child_user_esp;   // 分身專屬的 User ESP
    *(--kstack) = parent_eflags;    // EFLAGS (IF=1)
    *(--kstack) = parent_cs;        // User CS (0x1B)
    *(--kstack) = parent_eip;       // 回到 fork 呼叫的下一行

    // --- 階段 2：isr_common_stub 壓入的 Error Code 與 Int Number ---
    *(--kstack) = 0; // 假 Error Code
    *(--kstack) = 0; // 假 Int Number

    // --- 階段 3：isr_common_stub 的 pusha ---
    *(--kstack) = 0;   // EAX = 0 (我是小孩！)
    *(--kstack) = ecx; 
    *(--kstack) = edx; 
    *(--kstack) = ebx; 
    *(--kstack) = 0;   // ESP (pusha 會忽略)
    *(--kstack) = ebp; 
    *(--kstack) = esi; 
    *(--kstack) = edi; 

    // --- 階段 4：isr_common_stub 壓入的段暫存器 (關鍵！) ---
    *(--kstack) = 0x23; // DS
    *(--kstack) = 0x23; // ES
    *(--kstack) = 0x23; // FS
    *(--kstack) = 0x23; // GS

    // --- 階段 5：給 switch_task 的返回狀態 ---
    // 當 switch_task 執行 ret 時，我們要讓它跳到 isr_common_stub 的「下半段」去執行！
    // 我們需要宣告一個新的跳板：
    extern void fork_ret(); 
    *(--kstack) = (uint32_t) fork_ret;

    // switch_task 的 popa 與 popf
    for(int i = 0; i < 8; i++) *(--kstack) = 0;
    *(--kstack) = 0x0202; // IF=1

    child->esp = (uint32_t) kstack;

    // 將分身掛入排程佇列
    child->next = current_task->next;
    current_task->next = child;

    return child->id;
}

```

#### 3. 建立究極跳板 `fork_ret` (`asm/switch_task.S`)

因為我們的假堆疊現在變得和真正的中斷一模一樣（多了 `DS`, `ES`, `FS`, `GS`），我們不能再用 `task_return_stub` 退場了。我們需要一個專屬的跳板，負責把它們 `pop` 出來。

請在 **`asm/switch_task.S`** 最下方新增：

```assembly
; asm/switch_task.S
global fork_ret

; 當分身被排程器選中，switch_task 執行 ret 後，會跳來這裡！
; 這裡的堆疊長得跟 isr_common_stub 準備退出時完全一樣。
fork_ret:
    pop gs
    pop fs
    pop es
    pop ds
    popa
    add esp, 8
    iret

```

存檔，執行 `make clean && make run`。
