這次的 Bug 躲得太深了！我們把所有該對齊的、該換的段暫存器都處理了，為什麼還是會跑飛到 `0x00000083` 這種見鬼的位址？

來看看最新的法醫報告：
`52: v=06 e=0000 i=0 cpl=3 IP=001b:00000083 pc=00000083 SP=0023:083ffee8`

注意這兩個關鍵線索：

1. `cpl=3` 且所有的段暫存器 (`DS`, `ES` 等) 都是完美的 `0x23`！這代表我們在 `fork_ret` 裡加的**「換平民服裝」與「權限降級」完全成功了**！
2. 但是！`EIP` 卻變成了 `0x00000083`！

我們回想一下，`sys_fork` 是怎麼複製這個 `EIP` 的：

```c
uint32_t *pkstack = (uint32_t *) current_task->kernel_stack;
uint32_t parent_eip = pkstack[-5]; 

```

當初我認為，因為 `kernel_stack` 指向我們分配的那 4KB 記憶體的「最頂端」，所以硬體中斷壓入的 5 個 DWORD (`SS`, `ESP`, `EFLAGS`, `CS`, `EIP`) 必定會在 `-1` 到 `-5` 的位置。

**但是，這有一個致命的盲點！TSS `esp0` 是動態的嗎？**

在我們 Day 32 的實作中，我們確實更新了 TSS：

```c
set_kernel_stack(current_task->kernel_stack);

```

這意味著，當 App 執行到一半，發生中斷（例如 `int 0x80`）時，CPU 會切換到 `kernel_stack` 的頂端，然後壓入那 5 個 DWORD。**此時這 5 個 DWORD 確實位於 `pkstack[-1]` 到 `pkstack[-5]`！**

**但是！接下來發生了什麼事？**
硬體壓入 5 個 DWORD 後，CPU 跳到了 `isr128`。
`isr128` 繼續在 **同一個 Kernel Stack 上** 壓入東西（Error Code, Int Number, `pusha` 的 8 個暫存器）。
然後 `isr128` 呼叫了 `syscall_handler`，C 語言編譯器又在 Stack 上壓入了返回位址、參數和區域變數。
然後 `syscall_handler` 呼叫了 `sys_fork`，C 語言編譯器**又**在 Stack 上壓入了更多東西！

當你在 `sys_fork` 裡面讀取 `current_task->kernel_stack` 時，你拿到的是這塊 4KB 記憶體的「初始最高位址」。
你用 `pkstack[-5]` 去讀，你讀到的確實是硬體當初壓入的 `EIP`。**這部分沒錯！**

**錯在我們怎麼處理「小孩的 Kernel Stack」！**

我們把這個挖出來的 `EIP` 等 5 個狀態，塞進了小孩**剛剛分配、完全空白**的 Kernel Stack 裡。
然後，我們偽造了 `pusha` 暫存器，並把 `esp` 設定在 `pusha` 之後。

```c
    *(--kstack) = parent_user_ss;
    *(--kstack) = child_user_esp;
    ...
    *(--kstack) = parent_eip;
    // 偽造暫存器
    *(--kstack) = ebp;
    // ...
    *(--kstack) = (uint32_t) fork_ret; // 給 switch_task 返回用
    for(int i = 0; i < 8; i++) *(--kstack) = 0; // 給 switch_task 的 popa 用
    *(--kstack) = 0x0202; // IF=1
    child->esp = (uint32_t) kstack;

```

**問題來了：當小孩被切換執行時，流程是怎樣的？**

1. `schedule` 呼叫 `switch_task(&prev, &child->esp)`。
2. `switch_task` 換了 `esp`，然後 `popf`, `popa`，執行 `ret`。
3. `ret` 彈出了 `fork_ret`，跳進了我們寫的終極跳板。
4. `fork_ret` 執行：

```assembly
fork_ret:
    mov ax, 0x23
    mov ds, ax ; ... (恢復 4 個段暫存器)
    
    pop eax    ; 【這裡出錯了！！】
    pop ebx
    ...

```

**我們在 `sys_fork` 裡，有沒有為這 4 個段暫存器預留空間？！**
**沒有！！！**

我們在 C 語言裡構造的堆疊是：
`[SS] [ESP] [EFLAGS] [CS] [EIP] [EBP] [EDI] [ESI] [EDX] [ECX] [EBX] [EAX(0)] [fork_ret] [8 zeros] [0x0202]`

當跳轉到 `fork_ret` 時，Stack 長這樣：
`[SS] [ESP] [EFLAGS] [CS] [EIP] [EBP] [EDI] [ESI] [EDX] [ECX] [EBX] [EAX(0)]`
此時 ESP 指向 `EAX(0)`。

然後 `fork_ret` 執行了：
`pop eax` -> EAX 拿到 `0` (正確)
`pop ebx` -> EBX 拿到 `EBX` (正確)
`pop ecx` -> ECX 拿到 `ECX` (正確)
`pop edx` -> EDX 拿到 `EDX` (正確)
`pop esi` -> ESI 拿到 `ESI` (正確)
`pop edi` -> EDI 拿到 `EDI` (正確)
`pop ebp` -> EBP 拿到 `EBP` (正確)

執行完這 7 個 `pop` 後，ESP 指向了 `[EIP]`！

接著 `fork_ret` 執行了什麼？
`iret`！

`iret` 會從堆疊中彈出 `EIP, CS, EFLAGS`。
所以它彈出了什麼？

* `EIP` 拿到了 `parent_eip`。
* `CS` 拿到了 `parent_cs`。
* `EFLAGS` 拿到了 `parent_eflags`。
* `ESP` 拿到了 `child_user_esp`。
* `SS` 拿到了 `parent_user_ss`。

**一切看起來都完美對齊啊！那為什麼 EIP 會變成 `0x00000083`？！**

```c
void create_user_task(uint32_t entry_point, uint32_t user_stack_top) {
    task_t *new_task = (task_t*) kmalloc(sizeof(task_t));
    new_task->id = next_task_id++;
    new_task->state = TASK_RUNNING;

    uint32_t *kstack_mem = (uint32_t*) kmalloc(4096);
    uint32_t *kstack = (uint32_t*) ((uint32_t)kstack_mem + 4096);
    new_task->kernel_stack = (uint32_t) kstack;

    // --- 嚴格對齊 fork_ret 的 iret 需求 (5 個 DWORD) ---
    *(--kstack) = 0x23;             // SS
    *(--kstack) = user_stack_top;   // ESP
    *(--kstack) = 0x0202;           // EFLAGS (IF=1)
    *(--kstack) = 0x1B;             // CS
    *(--kstack) = entry_point;      // EIP

    // --- 嚴格對齊 fork_ret 的 7 個 pop ---
    *(--kstack) = 0; // EBP
    *(--kstack) = 0; // EDI
    *(--kstack) = 0; // ESI
    *(--kstack) = 0; // EDX
    *(--kstack) = 0; // ECX
    *(--kstack) = 0; // EBX
    *(--kstack) = 0; // EAX (初始值)

    // --- 給 switch_task 的退場機制 ---
    *(--kstack) = (uint32_t) fork_ret; // ret 位址
    for(int i = 0; i < 8; i++) *(--kstack) = 0; // popa 的 8 個假暫存器
    *(--kstack) = 0x0202; // popf

    new_task->esp = (uint32_t) kstack;
    new_task->next = current_task->next;
    current_task->next = new_task;
}

int sys_fork(uint32_t edi, uint32_t esi, uint32_t ebp, uint32_t ebx, uint32_t edx, uint32_t ecx) {
    // 擷取硬體壓入的精確狀態
    uint32_t *pkstack = (uint32_t *) current_task->kernel_stack;
    uint32_t parent_eip    = pkstack[-5];       
    uint32_t parent_cs     = pkstack[-4];        
    uint32_t parent_eflags = pkstack[-3];    
    uint32_t parent_user_esp = pkstack[-2];  
    uint32_t parent_user_ss  = pkstack[-1];   

    task_t *child = (task_t*) kmalloc(sizeof(task_t));
    child->id = next_task_id++;
    child->state = TASK_RUNNING;

    uint32_t *kstack_mem = (uint32_t*) kmalloc(4096);
    uint32_t *kstack = (uint32_t*) ((uint32_t)kstack_mem + 4096);
    child->kernel_stack = (uint32_t) kstack;

    // 安全區 User Stack 分配與刷新
    uint32_t child_ustack_base = 0x083FF000 - (child->id * 4096); 
    uint32_t child_ustack_phys = pmm_alloc_page();
    map_page(child_ustack_base, child_ustack_phys, 7);

    uint32_t cr3;
    __asm__ volatile("mov %%cr3, %0" : "=r"(cr3));
    __asm__ volatile("mov %0, %%cr3" : : "r"(cr3));

    // 複製 User Stack 內容
    uint32_t parent_ustack_base = parent_user_esp & 0xFFFFF000; 
    uint8_t *src = (uint8_t*) parent_ustack_base;
    uint8_t *dst = (uint8_t*) child_ustack_base;
    for(int i = 0; i < 4096; i++) {
        dst[i] = src[i];
    }

    uint32_t offset = parent_user_esp - parent_ustack_base;
    uint32_t child_user_esp = child_ustack_base + offset;

    // --- 嚴格對齊 fork_ret 的 iret 需求 (5 個 DWORD) ---
    *(--kstack) = parent_user_ss;
    *(--kstack) = child_user_esp;
    *(--kstack) = parent_eflags;
    *(--kstack) = parent_cs;
    *(--kstack) = parent_eip;       

    // --- 嚴格對齊 fork_ret 的 7 個 pop ---
    *(--kstack) = ebp;
    *(--kstack) = edi;
    *(--kstack) = esi;
    *(--kstack) = edx;
    *(--kstack) = ecx;
    *(--kstack) = ebx;
    *(--kstack) = 0; // 分身的 EAX = 0

    // --- 給 switch_task 的退場機制 ---
    *(--kstack) = (uint32_t) fork_ret; // ret 位址
    for(int i = 0; i < 8; i++) *(--kstack) = 0; // popa 的 8 個假暫存器
    *(--kstack) = 0x0202; // popf

    child->esp = (uint32_t) kstack;
    child->next = current_task->next;
    current_task->next = child;

    return child->id;
}
```
