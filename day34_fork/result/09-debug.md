這真的是 OS 開發中最令人挫敗但也最淬鍊心智的時刻了。

我們來看這次法醫報告最殘酷的部分：

```text
check_exception old: 0xffffffff new 0x6
    44: v=06 e=0000 i=0 cpl=3 IP=001b:00000083 pc=00000083 SP=0023:083ffee8 env->regs[R_EAX]=00000000

```

1. **Exception 0x06 (Invalid Opcode)**
2. 發生在 Ring 3 (`cpl=3`)
3. EIP 跑到了不可思議的 `0x00000083`！
4. 段暫存器一切正常 (`0x23` 和 `0x1B`)，EAX 也完美變成了 `0`（代表分身邏輯是對的）。

這表示什麼？
這表示我上一回合信誓旦旦跟你說的「統一且精確的堆疊對齊」，**還是對齊錯了！**
分身確實成功降落回 Ring 3，但它回錯了家，回到了位址 `0x83` 去執行垃圾指令。

為什麼？
因為我完全忽略了，我們呼叫 `sys_fork` 時，是透過 `syscall_handler`。而 `syscall_handler` 的參數宣告是：
`int sys_fork(uint32_t edi, uint32_t esi, uint32_t ebp, uint32_t ebx, uint32_t edx, uint32_t ecx)`

這裡有 **6 個參數**。
在 C 語言的呼叫慣例 (cdecl) 中，當 `syscall_handler` 呼叫 `sys_fork` 時，它會把這 6 個參數**壓入當前的堆疊**！這會讓 `current_task->kernel_stack` 指向的位址，被這 6 個參數（加上返回位址）給蓋掉！

所以當你在 `sys_fork` 裡面，天真地去讀取 `pkstack[-5]` 時，**你讀到的根本不是硬體壓入的 EIP，而是 C 語言剛壓入的參數或區域變數！**

### 🗡️ 絕對防彈解法：讓組合語言來代勞 (Raw ASM Stub)

我們不能再依賴 C 語言去猜測堆疊的狀態了。編譯器太聰明，也太不可控。
我們要寫一個**專屬於 `fork` 的中斷進入點 (Entry Point)**。當使用者呼叫 `fork` 時，我們不走傳統的 `syscall_handler`，而是直接跳到我們手寫的組合語言裡，第一時間把絕對乾淨的硬體狀態抓出來！

這是 Linux 早期核心（以及所有教科書 OS）處理 `fork` 的標準做法。請跟著我執行這三步「破釜沉舟」的終極重構：

#### 1. 打造專屬的 `sys_fork` 進入點 (`asm/switch_task.S`)

請打開 **`asm/switch_task.S`**，在 `fork_ret` 底下加入這個神級函式。
這個函式會作為我們 `fork` 的真正處理中心。它負責手工提取 IRET 幀，然後把它當作單一一個指標傳給 C 語言。

```assembly
[bits 32]
global switch_task
global fork_ret
global asm_do_fork   ; [新增]

; ... 前面的 switch_task 和 fork_ret 保留 ...

; ==========================================
; 【終極 Fork 進入點】直接從中斷處理器跳過來
; ==========================================
asm_do_fork:
    ; 此刻的堆疊長這樣：
    ; [ESP+16] SS
    ; [ESP+12] User ESP
    ; [ESP+8]  EFLAGS
    ; [ESP+4]  CS
    ; [ESP+0]  EIP 
    ; 這是絕對乾淨的硬體壓入狀態！

    ; 我們把現在的 ESP (也就是這 5 個 DWORD 的開頭位址) 當作參數，壓給 C 語言
    push esp

    ; 呼叫 C 語言的 do_fork_c (我們等一下寫)
    extern do_fork_c
    call do_fork_c

    ; 清理剛剛壓入的參數
    add esp, 4

    ; C 語言回傳的老爸 PID 會放在 EAX 裡
    ; 我們現在直接返回，把 EAX 帶回給老爸！
    ret

```

#### 2. 重寫 C 語言的 `fork` 邏輯 (`lib/task.c`)

現在，C 語言接收到的將是一個**保證指向 EIP** 的指標，再也沒有任何猜測的空間。

請打開 **`lib/task.c`**，把舊的 `sys_fork` 刪掉，換成這個 `do_fork_c`：

```c
// [修改] 絕對精準版的 Fork (由 asm_do_fork 呼叫)
int do_fork_c(uint32_t *iret_frame) {
    // iret_frame 是一個陣列，完美對應硬體壓入的 5 個 DWORD
    uint32_t parent_eip      = iret_frame[0];       
    uint32_t parent_cs       = iret_frame[1];        
    uint32_t parent_eflags   = iret_frame[2];    
    uint32_t parent_user_esp = iret_frame[3];  
    uint32_t parent_user_ss  = iret_frame[4];   

    task_t *child = (task_t*) kmalloc(sizeof(task_t));
    child->id = next_task_id++;
    child->state = TASK_RUNNING;

    uint32_t *kstack_mem = (uint32_t*) kmalloc(4096);
    uint32_t *kstack = (uint32_t*) ((uint32_t)kstack_mem + 4096);
    child->kernel_stack = (uint32_t) kstack;

    // 分配安全區的 User Stack 並刷新 TLB
    uint32_t child_ustack_base = 0x083FF000 - (child->id * 4096); 
    uint32_t child_ustack_phys = pmm_alloc_page();
    map_page(child_ustack_base, child_ustack_phys, 7);

    uint32_t cr3;
    __asm__ volatile("mov %%cr3, %0" : "=r"(cr3));
    __asm__ volatile("mov %0, %%cr3" : : "r"(cr3));

    // 複製 User Stack 內容
    uint32_t parent_ustack_base = parent_user_esp & 0xFFFFF000; 
    uint8_t *src = (uint8_t*) parent_ustack_base;
    uint8_t *dst = (uint8_t*) child_ustack_base;
    for(int i = 0; i < 4096; i++) {
        dst[i] = src[i];
    }

    uint32_t offset = parent_user_esp - parent_ustack_base;
    uint32_t child_user_esp = child_ustack_base + offset;

    // --- 嚴格對齊 fork_ret 的 iret 需求 (5 個 DWORD) ---
    *(--kstack) = parent_user_ss;
    *(--kstack) = child_user_esp;
    *(--kstack) = parent_eflags;
    *(--kstack) = parent_cs;
    *(--kstack) = parent_eip;       

    // --- 嚴格對齊 fork_ret 的 7 個 pop ---
    // (這裡不用管老爸的暫存器了，我們只關心 EAX=0)
    *(--kstack) = 0; // EBP
    *(--kstack) = 0; // EDI
    *(--kstack) = 0; // ESI
    *(--kstack) = 0; // EDX
    *(--kstack) = 0; // ECX
    *(--kstack) = 0; // EBX
    *(--kstack) = 0; // 分身的 EAX = 0！

    // --- 給 switch_task 的退場機制 ---
    extern void fork_ret();
    *(--kstack) = (uint32_t) fork_ret; // ret 位址
    for(int i = 0; i < 8; i++) *(--kstack) = 0; // popa 的假暫存器
    *(--kstack) = 0x0202; // popf

    child->esp = (uint32_t) kstack;
    
    // 掛入排程佇列
    child->next = current_task->next;
    current_task->next = child;

    // 回傳給老爸 PID
    return child->id;
}

```
