確認過 build-app.sh 修改了

```bash
#!/bin/bash

rm -f app.o my_app.elf

# 1. 編譯獨立的 User App (加入解除武裝的參數)
docker run --platform linux/amd64 --rm -v $(pwd):/workspace -w /workspace os-builder gcc -m32 -ffreestanding -nostdlib -fno-pie -fno-pic -fno-stack-protector -c app.c -o app.o

# 2. 連結 App
docker run --platform linux/amd64 --rm -v $(pwd):/workspace -w /workspace os-builder ld -m elf_i386 -Ttext 0x08048000 app.o -o my_app.elf
```

### 🔍 案發現場解密：為什麼會死在 `0xFFFFFFFC`？

我們來當一次系統法醫：

1. `EIP=0804804e`：CPU 當時正在執行 `my_app.elf` 裡面的程式碼。
2. `CR2=fffffffc`：這是 Page Fault 的案發地點。為什麼 CPU 會突然想去存取 `0xFFFFFFFC` 這個奇怪的高位址？

**破案關鍵：**
當你的 Ring 3 應用程式執行到 `sys_print` 裡面的 `int $0x80` 時，CPU 必須切換回 Ring 0 (核心模式) 來執行 Syscall。
為了安全起見，CPU **絕對不能**繼續使用 Ring 3 的堆疊 (`0x083FFF5C`)，它必須切換到 **Kernel Stack (核心堆疊)**，並且把 Ring 3 的狀態 (SS, ESP, EFLAGS, CS, EIP) push 上去保存。

CPU 要去哪裡找 Kernel Stack 的位址？答案是：查閱 **TSS 結構中的 `esp0` 欄位**。

因為我們在 Day 17 設定 TSS 時，可能忘記把 `esp0` 填上正確的位址（或者預設是 `0`）。
所以當 `int 0x80` 發生時，CPU 以為核心堆疊在 `0x00000000`。它往下減 4 bytes 準備 push 資料，就變成了 **`0xFFFFFFFC`**。這是一塊根本不存在/未映射的記憶體，於是當場觸發 Page Fault (`0xE`)，接著引發 Double Fault (`0x8`) 崩潰重開機！

### 🛠️ 終極修復：告訴 CPU 核心堆疊在哪裡

我們必須在跳入 Ring 3 之前，把當前 Kernel 的堆疊位址寫入 TSS 結構裡。

#### 第一步：在 GDT 模組中新增更新 TSS 的 API (`lib/gdt.c` & `lib/gdt.h`)

請打開 **`lib/gdt.c`**。在這個檔案中，你應該有一個用來定義 TSS 的結構體（通常命名為 `tss_entry` 或是 `tss`）。我們需要新增一個函式來更新它的 `esp0`：

```c
// 在 lib/gdt.c 檔案的下方新增這個函式：
// (請根據你在 Day 17 定義的 TSS 變數名稱，將 tss_entry 替換成正確的名稱)

void set_kernel_stack(uint32_t esp) {
    // 0x10 代表 Kernel Data Segment (你的 GDT 設定中，Data 段的 offset)
    tss_entry.ss0 = 0x10; 
    tss_entry.esp0 = esp; 
}

```

接著，打開 **`lib/gdt.h`**，把這個函式暴露出去：

```c
// 新增在 lib/gdt.h
void set_kernel_stack(uint32_t esp);

```

#### 第二步：在 Kernel 降級前，記錄當下的堆疊位址 (`lib/kernel.c`)

打開 **`lib/kernel.c`**，找到我們準備呼叫 `enter_user_mode` 的地方。
我們要在跳轉前一刻，把當前的 `esp` 抓出來，塞進 TSS 裡面！

```c
        kprintf("\nLoading ELF Application...\n");
        uint32_t entry_point = elf_load(real_app);
        
        if (entry_point != 0) {
            kprintf("Allocating User Stack and Dropping to Ring 3...\n\n");
            
            uint32_t user_stack_phys = pmm_alloc_page();
            map_page(0x083FF000, user_stack_phys, 7);
            uint32_t user_stack_ptr = 0x083FF000 + 4096;
            
            // --- [新增] 記錄 Kernel Stack 到 TSS ---
            uint32_t current_kernel_esp;
            __asm__ volatile("mov %%esp, %0" : "=r"(current_kernel_esp));
            set_kernel_stack(current_kernel_esp);
            // -------------------------------------

            extern void enter_user_mode(uint32_t entry_point, uint32_t user_stack);
            enter_user_mode(entry_point, user_stack_ptr);
        }

```
