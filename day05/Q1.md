
# Q1: 我跑出來結果的 Magic Number 怪怪的


哈哈，這是一個超級經典的 C 語言陷阱，你的觀察非常敏銳！看到截圖上印出 `0x./+.,///`，這不僅不是硬體壞掉，反而是 C 語言編譯器忠實執行指令的最佳證明。

這正是我們在打造底層系統時，最常遇到的**「有號整數 (Signed Integer) 溢位」**問題。

### 錯誤原因解析：為什麼會印出 `/` 與 `+`？

1. **隱藏的負數：** 我們傳入的 `3735928559` 換算成十六進位是 `0xDEADBEEF`。在 C 語言中，標準的 `int` 是 **32 位元的有號整數 (Signed)**，它的正數最大只能到 `2147483647`。所以 `3735928559` 存進 `int` 時，最高位的 `1` 會被當成負號，它實際上變成了 **`-559038737`**。
2. **負數的餘數：** 當我們的 `while` 迴圈對這個負數做 `value % 16` 時，C 語言算出來的餘數 `rem` 會是**負數**（例如 `-1`, `-5` 等）。
3. **ASCII 碼的推移：** 我們的轉換邏輯是 `rem + '0'`。字元 `'0'` 的 ASCII 碼是 `48`。
* 如果 `rem` 是 `-1`，`-1 + 48 = 47`，查 ASCII 表，`47` 正好就是 **`/`**！
* 如果 `rem` 是 `-5`，`-5 + 48 = 43`，查 ASCII 表，`43` 正好就是 **`+`**！



這完美解釋了為什麼會印出 `0x./+.,///`。至於為什麼 `0xB8000` (十進位是 `753664`) 能正常印出？因為它夠小，在 `int` 裡面依然是正數，所以餘數運算一切正常。

---

### 解決方案：引入無號整數 (Unsigned Int)

要解決這個問題，我們必須在處理十六進位時，強迫系統把它當作「無號整數 (Unsigned Integer)」來看待。

請修改 `kernel.c` 中的兩個地方：

#### 1. 更新 `itoa` 函式

我們在內部加入一個 `unsigned int` 變數來處理除法迴圈，讓負數可以直接以二補數的無號形式被處理：

```c
void itoa(int value, char* str, int base) {
    int i = 0;
    bool is_negative = false;
    unsigned int uvalue; // [新增] 用來做運算的無號整數

    if (value == 0) {
        str[i++] = '0';
        str[i] = '\0';
        return;
    }

    // 處理負數 (僅限十進位)
    if (value < 0 && base == 10) {
        is_negative = true;
        uvalue = (unsigned int)(-value);
    } else {
        // [關鍵] 如果是16進位，直接把位元當作無號整數看待 (完美對應記憶體的原始狀態)
        uvalue = (unsigned int)value;
    }

    // 逐位取出餘數 (改用 uvalue)
    while (uvalue != 0) {
        int rem = uvalue % base;
        str[i++] = (rem > 9) ? (rem - 10) + 'A' : rem + '0';
        uvalue = uvalue / base;
    }

    if (is_negative) {
        str[i++] = '-';
    }

    str[i] = '\0';
    reverse_string(str, i);
}

```

#### 2. 更新 `kprintf` 中的 `case 'x'`

取出參數時，也明確告訴 `va_arg` 我們要取的是一個無號整數：

```c
            case 'x': { // 十六進位整數
                // [關鍵] 改用 unsigned int 取出參數
                unsigned int num = va_arg(args, unsigned int);
                char buffer[32];
                terminal_writestring("0x");
                itoa(num, buffer, 16);
                terminal_writestring(buffer);
                break;
            }

```
