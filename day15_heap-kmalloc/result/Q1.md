
# Q1: 順利編譯過了，但是開起來之後，會一直 loop reboot，如圖。



哈哈，歡迎再次回到 Triple Fault（三重錯誤）的無限輪迴地獄！

這絕對不是你打錯了什麼，而是我們剛剛在 Day 15 的實作裡，不小心踩到了一個**我們在 Day 14 埋下的地雷**。

### 🔍 兇手調查：消失的 3GB 分頁表

讓我們來還原案發現場：

1. 在 `kheap.c` 裡，我們決定把 Kernel Heap 放在 `0xC0000000`（也就是 3GB 的位置）。
2. 我們呼叫了 `map_page(0xC0000000, phys_ptr, 3)`。
3. **【問題來了】** 回頭看我們 Day 14 寫的 `map_page`，它計算出來的目錄索引 (`pd_idx`) 會是 `0xC0000000 >> 22 = 768`。
4. 但是在 `map_page` 的 `if` 判斷裡，我們只認得 `pd_idx == 0` (0MB) 和 `pd_idx == 512` (2GB)。遇到 `768` 時，它印了一句錯誤訊息，然後直接 `return;`，**根本沒有映射成功！**
5. 下一秒，`kheap.c` 繼續往下執行，試圖把標頭寫入 `0xC0000000`：`heap_head->size = ...`。
6. CPU 發現這個虛擬位址沒有映射，立刻拋出 **Page Fault (第 14 號例外中斷)**。
7. 因為我們的 IDT 根本沒寫 14 號中斷的跳板，於是 Double Fault -> Triple Fault -> **QEMU 無限重啟**！

---

### 🛠️ 解決方案：為 3GB 建立專屬的分頁表

我們只需要在 `lib/paging.c` 裡面，多準備一張給 `0xC0000000` (索引 768) 用的分頁表就可以了。在真正的 OS 裡，這裡會動態產生分頁表，但為了維持簡單，我們一樣先手動加一張。

請打開 `lib/paging.c`，做以下四個小修改：

**1. 新增第三張分頁表：**
在檔案最上方宣告：

```c
uint32_t page_directory[1024] __attribute__((aligned(4096)));
uint32_t first_page_table[1024] __attribute__((aligned(4096))); // 0MB
uint32_t second_page_table[1024] __attribute__((aligned(4096)));// 2GB
uint32_t third_page_table[1024] __attribute__((aligned(4096))); // [新增] 給 3GB (0xC0000000) 用

```

**2. 初始化第三張表：**
在 `init_paging` 函式裡加入清零迴圈：

```c
    // 初始化第三張表 (全部設為不存在)
    for(int i = 0; i < 1024; i++) {
        third_page_table[i] = 0; 
    }

```

**3. 把第三張表掛載到目錄的 768 號位置：**
在 `init_paging` 函式裡加入這行：

```c
    page_directory[0] = ((uint32_t)first_page_table) | 3;
    page_directory[512] = ((uint32_t)second_page_table) | 3;
    page_directory[768] = ((uint32_t)third_page_table) | 3; // [新增] 3GB 目錄項

```

**4. 讓 `map_page` 認得 768 號目錄：**
在 `map_page` 函式裡修改 `if` 條件：

```c
    uint32_t* page_table;
    if (pd_idx == 0) {
        page_table = first_page_table;
    } else if (pd_idx == 512) {
        page_table = second_page_table;
    } else if (pd_idx == 768) {
        page_table = third_page_table; // [新增] 支援 3GB 映射
    } else {
        kprintf("Error: Page table not allocated for this address!\n");
        return;
    }

```

---

改完這四個地方後，你的 `map_page` 就有能力在 `0xC0000000` 施展魔法了！
