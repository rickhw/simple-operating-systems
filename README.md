# Course

你好！很高興能擔任你的導師。實作一個作業系統（OS）是軟體工程師最浪漫的終極挑戰。這不僅僅是寫程式，更是一場與硬體溝通、管理混亂、建立秩序的旅程。

既然你擁有深厚的開發背景（包含 SRE、架構師與 DevOps 經驗），我們不需要從基礎語法教起。這門課程將專注於 **x86_64 架構**（使用 QEMU 模擬器），結合 **C 語言**與少量 **Assembly**，並採用現代化的開發邏輯。

---

### 課程設計核心原則

* **從零開始 (From Scratch)：** 不依賴標準函式庫 (No standard library)，我們會自己寫 `printf`、`malloc` 等。
* **迭代開發：** 每一階段都要能編譯並在 QEMU 上運行，看到實質的回饋。
* **工具鏈：** 使用 `gcc-cross-compiler`、`nasm`、`ld` (Linker) 與 `make`。

---

### 60 天實作課程大綱

我將這 60 天劃分為六個衝刺階段（Sprints），每個階段 10 天。

#### 第一階段：啟動與核心基礎 (Day 1-10)

**目標：** 從 BIOS 跳轉到你的核心，並能在螢幕上印出 "Hello World"。

* **Day 1-3:** 撰寫 Multiboot2 標頭、編寫最小化的引導程式 (Bootstrap)。
* **Day 4-6:** 建立交叉編譯環境，設定 Linker Script 處理記憶體佈局。
* **Day 7-10:** 實作基礎 VGA 驅動（文字模式），封裝自己的 `kprint` 函式。

#### 第二階段：中斷處理與記憶體管理 (Day 11-20)

**目標：** 讓 CPU 能處理外界訊號，並建立記憶體分配器。

* **Day 11-14:** 設置 GDT (Global Descriptor Table) 與 IDT (Interrupt Descriptor Table)。
* **Day 15-17:** 實作實體記憶體管理 (PMM) — 位圖 (Bitmap) 演算法。
* **Day 18-20:** 實作虛擬記憶體管理 (VMM) — 分頁機制 (Paging) 與核心堆積 (Kernel Heap/kmalloc)。

#### 第三階段：多工處理與排程 (Day 21-30)

**目標：** 讓系統具備同時執行多個任務的能力。

* **Day 21-23:** 定義程序控制區塊 (PCB) 與上下文交換 (Context Switch)。
* **Day 24-27:** 實作排程器 (Scheduler) — 從簡單的 Round Robin 開始。
* **Day 28-30:** 實作核心執行緒 (Kernel Threads) 與同步機制 (Mutex/Spinlock)。

#### 第四階段：檔案系統與驅動程式 (Day 31-40)

**目標：** 能夠從磁碟讀取資料，並建立檔案階層。

* **Day 31-33:** 實作 IDE/ATA 磁碟驅動程式。
* **Day 34-37:** 實作虛擬檔案系統 (VFS) 抽象層。
* **Day 38-40:** 實作一個簡單的檔案系統（如 FAT16 或自定義的 SimpleFS）。

#### 第五階段：使用者模式與 System Calls (Day 41-50)

**目標：** 讓應用程式運行在受保護的空間，並與核心溝通。

* **Day 41-43:** 進入 User Mode (Ring 3) 與任務切換。
* **Day 44-46:** 實作系統呼叫 (Syscalls) 介面（如 `read`, `write`, `fork`, `exec`）。
* **Day 47-50:** 實作簡單的 Terminal 殼層 (Shell)，支援 `ls`, `cat`, `echo`。

#### 第六階段：圖形介面與工具擴充 (Day 51-60)

**目標：** 脫離文字模式，進入畫布與視窗世界。

* **Day 51-54:** 實作 VBE (VESA BIOS Extensions) 驅動，進入高解析度線性幀緩衝 (FrameBuffer)。
* **Day 55-57:** 撰寫基礎圖形函式庫（畫點、線、矩形、字體渲染）。
* **Day 58-60:** 實作一個簡單的視窗合成器 (Compositor) 與系統工具（如 `top` 顯示進程狀態）。

---

### 給你的建議與反饋

1. **調試是最大的挑戰：** 在 OS 開發中，你沒有 `gdb` 可以直接連。建議先學會如何使用 QEMU 的偵錯開關（如 `-d int` 查看中斷）以及使用 GDB 遠端連接 QEMU 核心。
2. **不要過度設計：** 在這 60 天內，目標是「理解概念」而非做出「下一個 Linux」。例如：檔案系統選 FAT16 就好，不要挑戰實作實時日誌系統。
3. **關於硬體：** 既然你使用的是 **MacBook Pro M1 Max**，你需要建立一個 x86_64 的 Docker 容器來進行編譯，或者使用虛擬機運行 Linux，因為 macOS 的 Linker 與 Linux 格式 (ELF) 不同。
4. **每天兩小時的分配：**
* **前 30 分鐘：** 閱讀理論（例如：今天要做 IDT，先看 Intel 手冊關於 Interrupts 的章節）。
* **後 90 分鐘：** 實作與 Debug。
