### 核心與硬體介面控制總表

#### 一、 權限與執行狀態控制 (Privilege & State)

| 硬體機制 | 核心的作用與配置時機 | 具體用途與運作原理 |
| --- | --- | --- |
| **Ring 0 / 3** <br>

<br>(特權等級) | **配置時機：** 建立 GDT 時設定，透過 `iret` 指令切換。<br>

<br>**管理對象：** CPU 指令執行權限。 | CPU 的硬體保全系統。在 Ring 0 時，Kernel 可以執行所有危險指令（如 `cli` 關中斷、寫入硬體 Port）；切換到 Ring 3 時，硬體會直接沒收這些危險指令的執行權限。 |
| **TSS** <br>

<br>(任務狀態段) | **配置時機：** 初始化 GDT 時掛載，並在每次任務切換前更新 `esp0`。<br>

<br>**管理對象：** CPU 跨權限跳躍時的安全緩衝區。 | 這是 CPU 規定的「逃生路線圖」。當 Ring 3 應用程式觸發中斷（如 Syscall 或 Timer）要跳回 Ring 0 時，硬體會看 TSS 裡面記錄的核心專用 Stack (`esp0`) 在哪，強制把指標切換過去，防止駭客用 Ring 3 的假 Stack 攻擊核心。 |

#### 二、 事件與中斷處理 (Interrupts & Events)

| 硬體機制 | 核心的作用與配置時機 | 具體用途與運作原理 |
| --- | --- | --- |
| **IDT** <br>

<br>(中斷描述符表) | **配置時機：** 開機初期 (`init_idt`)，將表格位址交給 `lidt` 指令。<br>

<br>**管理對象：** 所有硬體訊號與軟體異常的「路由表」。 | 當鍵盤被按下、除以零錯誤、或發生 Page Fault 時，CPU 硬體會立刻查這張表，決定要把控制權交給哪一段 Kernel C 程式碼（ISR）來擦屁股。 |
| **ISR** <br>

<br>(中斷處理常式) | **配置時機：** 與 IDT 綁定。執行於 Ring 0。<br>

<br>**管理對象：** 中斷發生時的案發現場保護。 | 這是組合語言寫的保鑣（如 `isr32`、`isr128`），負責第一時間 `pusha` 把所有暫存器存起來，執行完 Kernel 邏輯後，再 `popa` 原封不動還原，讓被打斷的程式毫無察覺。 |
| **Syscall** <br>

<br>(系統呼叫) | **配置時機：** 掛載於 IDT 的第 128 項 (`int 0x80`)，權限開給 Ring 3。<br>

<br>**管理對象：** Ring 3 對 Ring 0 的合法服務窗口。 | 應用程式無法自己讀取硬碟或印字，只能把參數放進暫存器，主動觸發 `int 0x80` 軟體中斷，喚醒 Kernel 來代勞。 |

#### 三、 記憶體與空間隔離 (Memory Management)

| 硬體機制 | 核心的作用與配置時機 | 具體用途與運作原理 |
| --- | --- | --- |
| **GDT** <br>

<br>(全域描述符表) | **配置時機：** 開機第一件事 (`init_gdt`)，透過 `lgdt` 寫入。<br>

<br>**管理對象：** 記憶體的「大區塊」劃分與基本權限。 | 這是 x86 最古老的保護機制，用來劃分 Code Segment (程式碼) 和 Data Segment (資料)。我們用它來定義 Ring 0 與 Ring 3 的大範圍地盤。 |
| **Paging** <br>

<br>(記憶體分頁) | **配置時機：** 取得記憶體大小後，設定 PD/PT，並修改 `CR3` 與 `CR0` 暫存器啟動。<br>

<br>**管理對象：** 虛擬位址到實體硬體的即時翻譯。 | Kernel 最強大的幻術。它讓每一個應用程式都以為自己擁有完整的 4GB 記憶體空間。Kernel 透過修改分頁表，動態決定某個虛擬位址要對應到哪條實體 RAM，甚至是設定該分頁「禁止 User 讀寫」，達成完美的程式隔離。 |
| **CR3 暫存器** | **配置時機：** 每次切換不同應用程式 (Context Switch) 時更新。<br>

<br>**管理對象：** 當前應用程式的「翻譯字典」首頁。 | 切換任務時，Kernel 只要把新的 Page Directory 位址塞進硬體 `CR3` 暫存器，整個記憶體宇宙就會在一瞬間切換成下一個應用程式的專屬視角。 |
