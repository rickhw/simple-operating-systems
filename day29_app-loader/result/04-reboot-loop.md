這絕對是這 29 天以來，最精彩、最硬核、也最值得裱框的一個 Bug！

你看著這堆看似無意義的十六進位數字，其實它們正在對你大喊：「**硬碟讀取失敗，我一路滑行撞牆啦！**」

我們來當一次系統級名偵探，破解這場不可思議的「EIP 死亡滑行 (EIP Slide)」事件：

### 🔍 案發現場的完美推理

請仔細看你 Dump 出來的三個關鍵數字：

1. **`EAX=08048056`**：這個數字眼熟嗎？這是截圖裡 `my_app.elf` 的 Entry Point！我們在 `user_mode.S` 裡，剛好把這個地址塞進了 `EAX`。
2. **`CR2=0804a000`**：Page Fault 發生的地點。
3. **`EIP=0804a000`**：CPU 當時正在**執行** `0x0804a000` 的指令。

**這三個數字拼湊出了一個驚人的真相：**
CPU 從 `0x08048056` (進入點) 開始執行。但是，因為某種原因，這段記憶體裡**全部都是 `0**`！
在 x86 組合語言中，機碼 `00 00` 翻譯成指令叫做 `add byte [eax], al`。
這句指令會把資料寫進 `EAX` 指向的記憶體，但**它絕對不會改變 `EAX` 本身的值**！

於是，CPU 就像踩到香蕉皮一樣，不斷執行 `00 00`，`EIP` 一路往下狂衝，而 `EAX` 永遠保持著 `08048056`。它滑過了整個 `.text` 區段，滑過了 `.rodata`，直到 `EIP` 撞上了 `0x0804a000` 這個我們從來沒有映射過的分頁邊界，然後「砰！」—— Page Fault 崩潰！

### 🤔 為什麼記憶體裡全是 0？

因為我們的 **ATA 硬碟驅動程式太快了**！
在 Day 21/22 時，我們只讀寫 1 個磁區 (512 bytes)，所以沒問題。但今天 `my_app.elf` 有 9KB，我們寫了一個 `for` 迴圈，連續狂叫硬碟交出 19 個磁區。
當第一個磁區讀完後，我們的迴圈「立刻」對硬碟發出讀取第二個磁區的指令 (`0x20`)。此時硬碟還在喘息，根本沒準備好，結果指令被吃掉，後續的讀取全部失效。
最後，`kmalloc` 給你的記憶體緩衝區裡，除了第一個磁區有 ELF 標頭外，剩下的程式碼區域全是 QEMU 初始化的 `0`。

---

### 🛠️ 終極修復：為 ATA 驅動加入「喘息時間」

我們必須在連續讀寫時，強迫 CPU 等待硬碟把狀態切換為 BSY (忙碌)，然後再等待它準備好資料。

請打開你的 **`lib/ata.c`**，用這個升級版覆蓋。我們加入了標準的 `400ns` 延遲（連續讀取 4 次狀態暫存器），並且在讀寫前後都嚴格檢查 BSY：

```c
#include "ata.h"
#include "utils.h"
#include "tty.h"

#define ATA_PORT_DATA       0x1F0
#define ATA_PORT_SECT_COUNT 0x1F2
#define ATA_PORT_LBA_LOW    0x1F3
#define ATA_PORT_LBA_MID    0x1F4
#define ATA_PORT_LBA_HIGH   0x1F5
#define ATA_PORT_DRV_HEAD   0x1F6
#define ATA_PORT_STATUS     0x1F7
#define ATA_PORT_COMMAND    0x1F7

// 等待硬碟不忙碌
static void ata_wait_bsy() {
    while (inb(ATA_PORT_STATUS) & 0x80);
}

// 等待硬碟準備好傳輸資料
static void ata_wait_drq() {
    while (1) {
        uint8_t status = inb(ATA_PORT_STATUS);
        if ((status & 0x80) == 0 && (status & 0x08) != 0) {
            break; 
        }
    }
}

// 產生約 400ns 的延遲，讓硬碟有時間更新狀態
static void ata_delay() {
    inb(ATA_PORT_STATUS);
    inb(ATA_PORT_STATUS);
    inb(ATA_PORT_STATUS);
    inb(ATA_PORT_STATUS);
}

void ata_read_sector(uint32_t lba, uint8_t* buffer) {
    ata_wait_bsy(); // [新增] 開始前先確認硬碟有空
    
    outb(ATA_PORT_DRV_HEAD, 0xE0 | ((lba >> 24) & 0x0F));
    outb(ATA_PORT_SECT_COUNT, 1);
    outb(ATA_PORT_LBA_LOW, (uint8_t) lba);
    outb(ATA_PORT_LBA_MID, (uint8_t)(lba >> 8));
    outb(ATA_PORT_LBA_HIGH, (uint8_t)(lba >> 16));
    outb(ATA_PORT_COMMAND, 0x20); // 讀取
    
    ata_delay();    // [新增] 給硬碟一點時間掛上 BSY 旗標
    ata_wait_drq(); // 等待資料準備好
    
    uint16_t* ptr = (uint16_t*) buffer;
    for (int i = 0; i < 256; i++) {
        ptr[i] = inw(ATA_PORT_DATA);
    }
    
    ata_wait_bsy(); // [新增] 讀完後確認硬碟收尾完成
}

void ata_write_sector(uint32_t lba, uint8_t* buffer) {
    ata_wait_bsy(); // [新增] 開始前先確認硬碟有空
    
    outb(ATA_PORT_DRV_HEAD, 0xE0 | ((lba >> 24) & 0x0F));
    outb(ATA_PORT_SECT_COUNT, 1);
    outb(ATA_PORT_LBA_LOW, (uint8_t) lba);
    outb(ATA_PORT_LBA_MID, (uint8_t)(lba >> 8));
    outb(ATA_PORT_LBA_HIGH, (uint8_t)(lba >> 16));
    outb(ATA_PORT_COMMAND, 0x30); // 寫入
    
    ata_delay();    // [新增] 給硬碟一點時間反應
    ata_wait_drq(); 
    
    uint16_t* ptr = (uint16_t*) buffer;
    for (int i = 0; i < 256; i++) {
        outw(ATA_PORT_DATA, ptr[i]);
    }
    
    outb(ATA_PORT_COMMAND, 0xE7); // Cache Flush
    ata_wait_bsy(); 
}

```
