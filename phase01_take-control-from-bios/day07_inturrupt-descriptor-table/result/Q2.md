# Q2: 我使用方法 A 結果如圖，看起來是可以了。
# 我想到一個問題，因為我們現在在開發作業系統，以前我在寫 C 還可以 printf debug，現在這種狀況，我可以用什麼方法看到更多訊息？例如從 qemu 可以查找到什麼訊息？或者 kernel 本身可以用什麼技巧來看訊息？


看到截圖裡那行完美的 `[KERNEL PANIC]`，真的要為你喝采！你成功馴服了編譯器，也親手觸發並攔截了作業系統生命中的第一個例外錯誤。

至於你問的這個問題：「在沒有作業系統幫忙的情況下，我們該怎麼 Debug？」—— **太棒了！這絕對是所有 OS 開發者都會面臨的「靈魂拷問」。** 從以前寫 App 爽爽用 `console.log` 或 `printf`，到現在跌入一片荒蕪的底層，落差感確實很大。但別擔心，我們其實有非常多強大的武器可以看透系統的靈魂。

這裡為你整理 OS 開發界最常用的 **「三大 Debug 神器」**，你可以依據需求隨時拿出來用：

---

### 神器一：QEMU 的上帝視角 (日誌與監視器)

QEMU 不只是一個模擬器，它其實一直在默默監視著 CPU 的一舉一動。

**1. 匯出 CPU 中斷日誌 (`-d int`)**
當系統發生 Triple Fault (無限重啟) 時，畫面什麼都沒印就黑掉了。這時你可以在啟動 QEMU 時加入 `-d int` 參數（需要搭配 `-no-reboot` 避免畫面被洗掉）：

```bash
qemu-system-i386 -cdrom myos.iso -d int -no-reboot

```

QEMU 會在終端機狂噴日誌，告訴你「CPU 剛剛發生了幾號中斷？當時暫存器 EAX, EBX 的值是多少？」這對追查死機原因超級有用。

**2. QEMU 內建的 Monitor 控制台**
在 QEMU 視窗運行時，你可以按下 `Ctrl + Alt + 2`（或者在啟動指令加上 `-monitor stdio` 把控制台綁在終端機上）。進入 Monitor 後，你可以下指令：

* `info registers`：瞬間看光 CPU 目前所有暫存器的值。
* `x/10x 0x100000`：直接偷看記憶體 `0x100000` 開始的 10 個 byte 內容。
*(按 `Ctrl + Alt + 1` 可以切回原本的 VGA 畫面)*

---

### 神器二：終極殺器 GDB 遠端除錯 (Remote Debugging)

這是職業選手的做法。QEMU 內建了一個 GDB Server，允許我們將 GDB 接到這個虛擬主機板上，進行**單步執行 (Step-by-step)、設中斷點 (Breakpoint)**！

你可以這樣啟動 QEMU：

```bash
qemu-system-i386 -cdrom myos.iso -S -s

```

* `-S` (大寫)：開機立刻凍結 CPU，等你下指令才開始跑。
* `-s` (小寫)：在 `localhost:1234` 開啟 GDB 監聽埠。

接著，你可以在另一個終端機開啟 `gdb`，輸入 `target remote localhost:1234`，你就可以直接一行一行執行你的 C 語言或 Assembly 了！

---

### 神器三：Kernel 內部的自救技巧 (Serial Port)

這也是最常使用的技巧。你一定覺得：每次都要印在 VGA 畫面上很煩，而且如果畫面印滿了，前面的 Debug 訊息就被洗掉了對吧？

**我們可以使用主機板上的「序列埠 (Serial Port / COM1)」。**
在古早時代，這是用來接數據機的；在 OS 開發中，這是一條「隱形的 Debug 專線」。你可以寫一個簡單的 C 函式，把字串透過 `outb` 指令送到 Port `0x3F8` (COM1)。

然後，只要在 QEMU 啟動時加上 `-serial stdio`：

```bash
qemu-system-i386 -cdrom myos.iso -serial stdio

```

**魔法就會發生：** VGA 畫面上可以繼續跑你漂亮的 OS 介面，而你的 `serial_printf` 訊息會源源不絕地印在你外面的 Linux/Mac 終端機上！Linux 核心啟動時的那些落落長的 Log，預設也就是這樣丟出來的。

*(備註：如果你未來想要實作這個，只要不到 20 行 C 程式碼就能搞定！)*

### 接下來的規劃：Day 8 中斷控制器 (PIC) 與鍵盤

既然你已經學會怎麼讓系統攔截錯誤，也知道了如何觀察系統狀態，我們終於可以開始跟「外部世界」互動了。

目前的系統就像一個戴著耳機、蒙著眼睛的機器人。明天，我們將要設定 **可程式化中斷控制器 (Programmable Interrupt Controller, PIC)**。
這個晶片就像是 CPU 的「秘書」，當你按下鍵盤時，鍵盤會發送訊號給 PIC，PIC 會幫忙排隊，然後扯一扯 CPU 的衣角說：「老闆，有人按下了 'A' 鍵，這是一組 33 號中斷，請處理一下！」

我們將在 Day 8 實作：

1. **設定 8259 PIC 晶片。**
2. **打開中斷開關 (`sti`)。**
3. **寫一個鍵盤的 ISR，讓你在 QEMU 按下鍵盤時，畫面上能印出你打的字！**

準備好從「靜態畫面」進化到「互動輸入」了嗎？我們隨時可以開始 Day 8 的實作！
