### 🤯 1. 你創造了「孫子」行程 (Inception 般的套娃)

```text
SimpleOS> exit

SimpleOS> exit
Goodbye!
[PARENT] Child has finished! I am back in control.
```
為什麼你會退出「兩次」，而且兩次都觸發了老爸的等待喚醒？
因為你剛才的操作流程其實是：
1. 原本的 Shell (老爸) 執行 `run` 創造了子 Shell。
2. 你在**子 Shell 裡面又執行了一次 `run`**，創造了「孫子 Shell」！
3. 當你輸入第一次 `exit`，孫子死掉，喚醒了子 Shell。
4. 當你輸入第二次 `exit`，子 Shell 死掉，才終於喚醒了最原始的老爸 Shell！

你的作業系統完美扛住了**多層次的行程嵌套 (Nested Processes)**，這證明了我們底層的堆疊隔離、記憶體分頁、以及 `registers_t` 的狀態保存簡直無懈可擊！


### 💤 2. 為什麼會瘋狂洗版 CPU Idle？

當你輸入第三次 `exit` 離開最原始的老爸 Shell 時，就進入了第二張截圖 (`14.01.45.png`) 的畫面。這完全是**預期中的正確行為**！

這背後的邏輯非常迷人：
* 當最後一個 Shell 呼叫 `sys_exit()` 時，系統內所有的任務都變成了 `TASK_DEAD`。
* 我們的 `schedule()` 發現沒有任何活著的任務了，於是觸發了安全網，印出 `[Kernel] No runnable tasks. CPU Idle.`，然後執行 `hlt` 把 CPU 關掉進入省電模式。
* **那為什麼會洗版？** 因為硬體的時鐘中斷 (Timer IRQ0) 依然以每秒上百次的速度在跳動！
* 每「滴答」一次，CPU 就會被強迫醒來執行 `timer_handler` -> 呼叫 `schedule()` -> 發現還是沒人活著 -> **再印一次 Idle 訊息** -> 繼續 `hlt` 睡覺。

這其實就是現代作業系統中 **「Idle Task (空閒任務)」** 的雛形。在真實的 Linux 或 Windows 裡，當系統沒事做時，其實就是一個隱藏的 PID 0 (Idle Process) 在不斷執行 `hlt`。

### 📝 Day 35 核心觀念與 Debug 總結

1. **C 語言與組合語言的邊界 (Inline ASM 的危險)：**
   * **雷點：** 我們原本把 `child_ret_stub` 寫在 C 語言裡，並用 `__asm__ volatile` 強制修改 `EBP`。但 C 語言編譯器在處理函式參數時，非常依賴 `EBP`。我們手動竄改它，直接導致了編譯器找不到參數，引發 Page Fault。
   * **解法：** 將涉及暫存器底層恢復的跳板程式，**完全移到純組合語言 (`switch_task.S`)** 中實作，徹底隔絕編譯器的干擾。

2. **環狀鏈結串列 (Circular Linked List) 的無窮迴圈：**
   * **雷點：** 在 `schedule()` 尋找和清理死掉的任務時，我們用的 `while (next_node != curr)` 終止條件，因為裡面有 `curr = next_node` 導致指標同時往前跑，變成了永遠追不到的「貪食蛇」，讓 Kernel 卡死在排程器裡。
   * **解法：** 必須固定一個「起點」（例如 `current_task`），讓迴圈條件變成 `while (next_node != current_task)`，確保它只會繞整整一圈。

3. **Kernel 任務的功成身退：**
   * **雷點：** 系統看起來卡死不理鍵盤，是因為 Kernel 任務 (`main_task`) 在啟動 Shell 後，進入了 `while(1) { hlt; }` 迴圈。當排程器切換回 Kernel 時，它就卡在那裡不處理新的事情了。
   * **解法：** 讓 Kernel 任務在完成初始設定並喚醒第一個 User App 後，執行 `exit_task()` 把自己殺掉。把整個舞台 100% 交給 User Space 的應用程式。

4. **Fork-Exec-Wait 模型的完整閉環：**
   * **`fork`**：精準複製老爸的記憶體，並透過「外科手術」修補 EBP 鏈結，創造出完美的平信世界分身。
   * **`exec`**：無情地清空 User Stack，將 EIP 指向新的 ELF 程式進入點，實現靈魂轉移。
   * **`wait` & `exit`**：加入 `TASK_WAITING` 狀態，讓老爸主動陷入沉睡讓出 CPU，直到小孩死亡 (`exit`) 時負責把老爸搖醒，完美解決畫面與鍵盤的搶佔問題。
