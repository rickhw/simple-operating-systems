在作業系統的設計中，實作 `pwd` (Print Working Directory) 通常有兩種截然不同的切入角度。一種是「**偷吃步（記路法）**」，另一種則是「**正統 UNIX 魔法（爬樹法）**」。

我們來深度解析這兩種角度的優缺點，你可以決定你想挑戰哪一種！

---

### 角度一：偷吃步「記路法」 (String Caching)

這是最直觀、最容易實作的方法。
既然 Process 是透過 `cd` 一步一步走進去的，我們何不讓 Process 自己記住走過的路？

* **實作概念**：
    1. 在 `task_t` 裡面新增一個陣列，例如 `char cwd_path[256];`。
    2. 當行程剛建立時，預設為 `strcpy(cwd_path, "/");`。
    3. 每次呼叫 `sys_chdir` (切換目錄) 成功時，就去修改這串字串。
        * 如果是 `cd folder1`，就把字串接起來變成 `/folder1`。
        * 如果是 `cd ..`，就用字串處理的技巧，把最後一個 `/` 後面的東西砍掉。
    4. 當 User App 呼叫 `sys_getcwd` (獲取當前路徑) 時，Kernel 直接把這串字串複製給它就好。
* **優點**：執行速度極快，不需要讀取硬碟，實作簡單。
* **缺點**：
    * 字串處理很囉嗦（要處理 `cd a/b/c`、`cd ../../` 這種複雜情況）。
    * **最致命的缺點**：如果未來你實作了 `mv` (重新命名資料夾)，例如把 `folder1` 改名成 `folder2`，但某個背景程式的 `cwd_path` 還記著 `/folder1`，它的路徑就變成「死路」了！這在 OS 裡稱為 Stale Path。

### 角度二：正統 UNIX 魔法「爬樹法」 (Reverse Resolution)



這是真正的大神做法，也是現代 Linux/UNIX 尋找 `pwd` 的底層原理。
在真正的 UNIX 哲學中，**「路徑」不是被記下來的，而是「反向推導」出來的！**

因為我們昨天已經在每個資料夾裡面塞入了 `.` (自己) 和 `..` (老爸)，我們就可以利用這兩個幽靈檔案來「認祖歸宗」！

* **實作概念**：
    1. **「我是誰？」**：給定現在的 `cwd_lba`。如果它是 `1`（根目錄），那就直接回傳 `/`。
    2. **「找老爸」**：如果不是根目錄，我們就讀取 `cwd_lba` 這個目錄，尋找檔名叫 `..` 的項目，取得**老爸的 LBA (Parent LBA)**。
    3. **「問老爸我的名字」**：接著，我們讀取老爸的目錄 (`Parent LBA`)，在裡面尋找哪一個項目的 `start_lba` 剛好等於我們的 `cwd_lba`。找到後，那個項目的 `filename` 就是我們這個資料夾的真正名字！
    4. **「往上爬」**：把找到的名字存起來，然後把目前的 LBA 變成老爸的 LBA，重複步驟 2 和 3，直到我們撞到根目錄 (`LBA == 1`) 為止。
    5. **「反轉字串」**：因為我們是從底層往上找的（例如先找到 `sub`，再找到 `folder1`），最後把這些名字反向組裝起來，加上斜線，就會得出 `/folder1/sub`！

* **優點**：這才是真正的檔案系統！它永遠 100% 準確，不怕資料夾被改名，也不需要 Process 浪費記憶體去記字串。
* **缺點**：實作難度較高，需要頻繁讀取硬碟（但這對學習 OS 架構來說是絕佳的練習）。

---

### 你可以怎麼切入？

如果你想挑戰 **角度二 (爬樹法)**，你需要準備以下武器：

1.  **在 `lib/simplefs.c` 實作一個底層函數**：
    ```c
    // 概念：給定一個目錄 LBA，反推它的完整路徑並寫入 buffer
    void simplefs_getcwd(uint32_t current_lba, char* out_buffer);
    ```
2.  **新增一個系統呼叫 `sys_getcwd`**：
    負責呼叫 `simplefs_getcwd(current_task->cwd_lba, buffer)`。
3.  **在 Shell (`shell.c`) 中加入內建指令 `pwd`**：
    （跟 `cd` 一樣，`pwd` 通常也是 Shell 的內建指令，因為它緊密依賴當前行程的狀態。）
    ```c
    else if (strcmp(cmd, "pwd") == 0) {
        char path_buf[256];
        getcwd(path_buf); // 呼叫 syscall
        printf("%s\n", path_buf);
    }
    ```
    *（或者你想把 `pwd` 寫成一個獨立的 `pwd.elf` 也可以，因為它只需要透過 syscall 去問 Kernel 自己在哪裡就好。）*

你想走「輕鬆寫意的字串記路法」，還是挑戰「硬核浪漫的 UNIX 爬樹法」呢？選定方向後，我們可以一起把這塊拼圖給補上！😎
