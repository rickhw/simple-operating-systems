沒問題！在作業系統開發的初期，`boot.S` 是銜接硬體（或 Bootloader）與高階語言（如 C 語言）最關鍵的一橋樑。這段代碼使用的是 **NASM 語法**，針對 x86 架構。

我們將這段代碼拆解為語法邏輯與指令集兩個部分來複習。

---

## 第一部分：語法與指示字 (Directives) 說明

這些並非 CPU 執行的指令，而是給 **組譯器 (Assembler)** 的指令，告訴它如何組織產出的二進位檔。

### 1. 常數定義 (`equ`)
* **語法：** `NAME equ expression`
* **說明：** 類似 C 語言的 `#define`。它不會佔用最終編譯出來的記憶體空間，只是在組譯過程中將 `NAME` 替換為數值。
* **範例：** `FLAGS equ MBALIGN | MEMINFO` 只是計算出一個數值供後面使用。

### 2. 資料定義 (`dd`, `resb`)
* **`dd` (Define Double-word):** 配置 4 Bytes (32 bits) 的空間並寫入初始值。在 Multiboot Header 中，這用來固定放置 Magic Number。
* **`resb` (Reserve Byte):** 保留指定數量的位元組空間，但不賦予初始值。這通常用在 `.bss` 區段。
    * `resb 16384`：保留了 16 KB 給堆疊（Stack）使用。

### 3. 區段控制 (`section`, `align`)
* **`section .name`:** 將代碼或資料分組。
    * `.multiboot`: 特殊區段，必須放在檔案的最開頭，讓 Bootloader (GRUB) 能夠一眼看到。
    * `.bss`: 存放未初始化的全域變數，特點是不佔用硬碟空間，只在程式載入記憶體時分配。
    * `.text`: 存放實際要執行的 CPU 指令。
* **`align n`:** 強制下一個位址必須是 `n` 的倍數。
    * `align 4`: 為了符合 Multiboot 規範。
    * `align 16`: 為了符合 x86 堆疊對齊的要求（效能與相容性考量）。

### 4. 符號宣告 (`global`, `extern`)
* **`global _start`:** 告訴連結器 (Linker)，`_start` 這個標籤是公開的，這通常是整個程式的進入點。
* **`extern kernel_main`:** 告訴組譯器，`kernel_main` 這個符號在別的檔案（通常是 C 檔案）裡，現在先不用管它的具體位址，等連結時再處理。

## 第二部分：特有指令集 (Instructions) 說明

這部分是 CPU 真正會執行的動作。



### 1. 堆疊與暫存器操作
* **`mov esp, stack_top`:**
    * **作用：** 初始化堆疊指標（Stack Pointer）。
    * **細節：** 在 x86 中，堆疊是**由高位址向低位址增長**。所以我們把 `esp` 指向 `stack_top`（16KB 空間的末端）。
* **`push ebx / eax`:**
    * **作用：** 將暫存器的值壓入堆疊。
    * **目的：** 這是為了符合 **C Calling Convention (C 呼叫慣例)**。在呼叫 `kernel_main(magic, addr)` 時，參數必須由右至左壓入堆疊。
* **`add esp, 8`:**
    * **作用：** 歸還堆疊空間。
    * **說明：** 剛才壓入了兩個 4-byte 的參數，呼叫完 C 函式後，我們手動將指標移回去，等於「清空」了這兩個參數。

### 2. 流程控制與系統指令
* **`call kernel_main`:**
    * 將目前的 `eip` (指令指標) 壓棧，並跳轉到 C 語言定義的函式位置。
* **`cli` (Clear Interrupts):**
    * **作用：** 關閉所有可遮蔽的中斷。
    * **情境：** 當 Kernel 結束或發生嚴重錯誤時，我們不希望 CPU 被任何硬體中斷干擾。
* **`hlt` (Halt):**
    * **作用：** 讓 CPU 進入休眠狀態，直到下一個中斷發生。
    * **優點：** 比起 `jmp $` (無限迴圈)，`hlt` 更加省電且不會讓 CPU 核心溫度飆升。
* **`jmp .hang`:**
    * 如果 `hlt` 因為某種原因被喚醒，這行指令會讓它再次跳回 `.hang` 標籤，確保 CPU 永遠停留在這個區域。

### 補充：為什麼要用這兩個參數？
根據 Multiboot 規範，當 Bootloader 完成任務跳轉到你的 `_start` 時：
1.  `eax` 必定存放 `0x2BADB002`（確認是從合法的 Bootloader 啟動）。
2.  `ebx` 存放一個結構體位址，裡面包含記憶體大小、開機磁碟資訊等重要數據。
這就是為什麼我們需要 `push ebx` 和 `push eax` 後才呼叫 C 代碼的原因。

這是一個非常棒的問題！對於從高階語言（如 C/C++ 或 Java）轉到組合語言的人來說，最容易搞混的就是**暫存器 (Registers)** 與**變數 (Variables)** 的區別。

簡單來說：**暫存器不是變數，它們是 CPU 內部「內建」的極速儲存單元。** 你的 CPU 裡就只有那幾個固定名稱的暫存器，不像變數可以隨意命名。

---

## 暫存器的全名與由來

在 x86（32 位元）架構中，暫存器名稱開頭的 **「E」** 代表 **Extended (延伸)**。這是因為早期的 8086 CPU 是 16 位元的，後來升級到 32 位元時，為了相容性，就把原有的暫存器加長了。

### 1. 通用暫存器 (General Purpose Registers)
這類暫存器就像是 CPU 的「草稿紙」，可以用來做運算或傳遞參數。

| 簡寫 | 全名 | 主要用途 |
| :--- | :--- | :--- |
| **EAX** | **E**xtended **A**ccumulator Register | **累加器**。主要用於運算結果與函式傳回值。 |
| **EBX** | **E**xtended **B**ase Register | **基底暫存器**。常用於指向記憶體位址的基底。 |
| **ECX** | **E**xtended **C**ounter Register | **計數暫存器**。常用於迴圈（Loop）的計數。 |
| **EDX** | **E**xtended **D**ata Register | **資料暫存器**。常用於乘除法運算或 I/O 埠操作。 |

### 2. 指標暫存器 (Pointer Registers)
這類暫存器非常特殊，它們專門用來管理**堆疊 (Stack)**。

* **ESP (Extended Stack Pointer):** * **全名：** 延伸堆疊指標。
    * **作用：** 它永遠指向目前堆疊的最頂端（最後一個被推入資料的位置）。
* **EBP (Extended Base Pointer):** * **全名：** 延伸基底指標。
    * **作用：** 通常用來固定在一個函式的起點，方便存取區域變數。



---

## 為什麼 `boot.S` 要用到這三個？

在你的 `boot.S` 中，這三者各司其職：

1.  **`mov esp, stack_top`**: 
    你要 CPU 開始工作前，必須先告訴它「你的堆疊在哪裡」。我們把事先畫好的記憶體位址（`stack_top`）塞進 **ESP**，CPU 才知道以後 `push` 東西要往哪放。
    
2.  **`push eax` 與 `push ebx`**:
    根據 **Multiboot 規範**，Bootloader（如 GRUB）在跳轉到你的程式碼那一瞬間，會預先在 **EAX** 放一個魔術數字，在 **EBX** 放一個結構體的記憶體指標。
    * 我們把這兩個「暫存器裡的值」壓入堆疊，是為了讓接下來呼叫的 **C 語言函式** 能從堆疊裡把這兩個值當作參數讀出來。

## 暫存器 vs. 變數 的直觀比喻

想像你在做木工：
* **變數 (RAM/Memory)**：像是你後方的大材料架，東西很多，但你要走過去拿才拿得到。
* **暫存器 (Registers)**：像是你**手上的工具**（或掛在腰帶上的工具）。你一次只能拿這麼幾件，但你伸手就能用，速度極快。

> **小知識：** 雖然我們寫的是 `EAX` (32 bits)，但你依然可以只存取它的低 16 位元（稱為 `AX`），或是把 `AX` 再切成高 8 位元 (`AH`) 與低 8 位元 (`AL`)。這就是 x86 為了長年相容性所留下的痕跡。
