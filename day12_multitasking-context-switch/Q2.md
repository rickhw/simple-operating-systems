
### Simple OS 多工排程 (Task Switch) 終極流程圖

這張流程圖整合了我們昨天搬家後的 `lib/`、`asm/` 架構，以及你要求加入的 Ring 特權級別與 ISR 跳板邏輯。

```mermaid
flowchart TD
    %% 定義節點樣式
    classDef clang_r3 fill:#cfe2f3,stroke:#6fa8dc,stroke-width:2px,color:#000,stroke-dasharray: 5 5
    classDef clang_r0 fill:#cfe2f3,stroke:#6fa8dc,stroke-width:2px,color:#000
    classDef asm fill:#f9d0c4,stroke:#e06666,stroke-width:2px,color:#000
    classDef hw fill:#fff2cc,stroke:#d6b656,stroke-width:2px,color:#000

    %% 使用者任務 (這部分我們還沒實作，用虛線表示概念)
    subgraph UserSpace ["使用者空間 (Ring 3 - 低特權)"]
        UserAppA["「平行宇宙 A」應用程式"]:::clang_r3
    end

    %% CPU 攔截中斷
    UserAppA -- "程式跑完 10ms，發生例外或中斷" --> CPU_Detect

    subgraph CPUHard ["CPU 硬體保護機制"]
        CPU_Detect{{"CPU 偵測到中斷\n(例外、Timer 或鍵盤)"}}:::hw
        CPU_Detect -- "查 IDT。如果此時在 Ring 3..." --> StackSwitch
        StackSwitch["1. 硬體強制切換 `esp` 到預設安全的核心 Stack"]:::hw
        StackSwitch --> CPU_Jump
        CPU_Jump["2. CPU 跳躍至 IDT 聯絡簿指定的組合語言跳板"]:::hw
    end

    %% 核心空間處理 (這就是我們這幾天寫的 Code)
    subgraph KernelSpace ["核心空間 (Ring 0 - 最高特權)"]
        subgraph ASM_Lib ["asm/interrupts.S (組合語言跳板 ISR)"]
            ISR33["第 33 號中斷跳板"]:::asm
        end

        subgraph C_Lib ["lib/ (C 語言核心處理器)"]
            handlers["lib/timer.c\nlib/keyboard.c"]:::clang_r0
            scheduler["lib/task.c (排程器 - OS 大腦)"]:::clang_r0
        end

        subgraph TaskData ["任務資料結構 TCB"]
            TCBA["[Task A]\nesp = 0x10..."]:::clang_r0
            TCBB["[Task B]\nesp = 0x20..."]:::clang_r0
        end
    end

    %% 中斷跳板與 C 的關係 (ASM 呼叫 C)
    CPU_Jump -- "跳到" --> ISR33
    ISR33 -->|"1. pusha (保鑣保存 A 暫存器到 A 的 Stack)"| ISR33
    ISR33 -->|"2. 呼叫 C 處理函式"| handlers
    handlers -->|"處理完資料"| handlers
    handlers -->|"3. 呼叫排程器大腦"| scheduler

    %% 魔術發生：任務切換
    scheduler -->|"4. 排程器大腦覺得「該換 B 跑了」\n(這就是 Context Switch)"| scheduler
    scheduler -.->|"5. 從 TCBA 儲存 ESP\_A\n從 TCBB 載入 ESP\_B (偷天換日魔法)"| CPUHard
    
    scheduler -->|"6. 排程結束"| handlers
    handlers -->|"7. 回到跳板準備返回"| ISR33

    %% 從核心返回
    subgraph CPUReturn ["CPU 硬體恢復機制"]
        ReturnHandler["1. popa (恢復當前任務 B 的暫存器)"]:::asm
        ReturnHandler --> CPU_iret
        CPU_iret["2. iret (恢復中斷前的 eip、Ring 等狀態)"]:::hw
    end

    ISR33 -->|"執行返回程式碼"| ReturnHandler
    CPU_iret -- "中斷返回。因為載入了 B 的狀態..." --> UserAppB

    subgraph UserSpaceB ["使用者空間B (Ring 3)"]
        UserAppB["「平行宇宙 B」應用程式復活！"]:::clang_r3
    end

```

這張圖完美呈現了 CPU 是怎麼在中斷發生時從 Ring 3 換到安全的核心 Stack (Ring 0 ISR)，保存舊任務狀態，排程器介入「偷天換日」載入新任務 Stack Pointer，最後 `iret` 完美復活另一個宇宙的過程。

現在系統的骨架都已經模組化得很漂亮了，你也掌握了系統的核心名詞清單。
