# Course

## 課程設計核心原則

* **從零開始 (From Scratch)：** 不依賴標準函式庫 (No standard library)，我們會自己寫 `printf`、`malloc` 等。
* **迭代開發：** 每一階段都要能編譯並在 QEMU 上運行，看到實質的回饋。
* **工具鏈：** 使用 `gcc-cross-compiler`、`nasm`、`ld` (Linker) 與 `make`。

## Simple OS 60 天實作課程大綱 (更新版)

本專案將 60 天的開發旅程劃分為六個衝刺階段（Sprints）。目前已順利完成前兩個階段，成功建立具備記憶體分頁、特權階級隔離與執行檔載入能力的現代核心骨架。

### 第一階段：啟動與核心骨架 (Day 1-10) [✅ 已完成]

**目標：** 從 BIOS 接管控制權，建立中斷與基礎輸出能力。

* **Day 1-4:** GRUB Multiboot 啟動機制、VGA 文字模式與 `kprint` 基礎輸出實作。
* **Day 5-8:** GDT (全域描述符表)、IDT (中斷描述符表)、ISR 中斷跳板與 PIC 控制器。
* **Day 9-10:** 鍵盤驅動與 Timer (PIT) 基礎中斷處理，完成硬體事件攔截。

### 第二階段：記憶體與特權階級大挪移 (Day 11-20) [✅ 已完成]

**目標：** 建立現代記憶體管理機制，並成功在 Ring 3 執行外部應用程式。

* **Day 11-12:** 任務控制區塊 (TCB) 與基礎 Context Switch (上下文切換)。
* **Day 13-15:** 實體記憶體管理 (PMM)、虛擬記憶體分頁 (Paging)、核心堆積分配器 (`kmalloc`)。
* **Day 16-17:** 系統呼叫 (Syscall, `int 0x80`) 與 TSS 設定，防護降級至 User Mode (Ring 3)。
* **Day 18-20:** ELF 執行檔解析器、GRUB Multiboot 模組接收、虛實記憶體映射與真實 App 載入。

### 第三階段：儲存裝置與檔案系統 (Day 21-30) [🚀 現正進行中]

**目標：** 脫離 GRUB 模組依賴，讓系統具備讀取磁碟與檔案階層的能力。

* **Day 21-23:** ATA/IDE 硬碟驅動程式實作 (透過 PIO 模式與硬體 Port 溝通讀寫磁區)。
* **Day 24-27:** 虛擬檔案系統 (VFS) 抽象層設計，統一介面。
* **Day 28-30:** 實作精簡版檔案系統 (如 FAT16/FAT32 或 SimpleFS)，支援 `open`, `read`, `close` 系統呼叫。

### 第四階段：行程管理與多工排程進階 (Day 31-40)

**目標：** 完善應用程式的生命週期管理與資源競爭保護。

* **Day 31-33:** 實作完整的 `fork`, `execve`, `exit`, `wait` 系統呼叫。
* **Day 34-37:** 排程器 (Scheduler) 升級（從簡單的 Round Robin 演進到優先級排程）。
* **Day 38-40:** 核心同步機制 (Spinlock, Mutex, Semaphore) 與基礎行程間通訊 (IPC)。

### 第五階段：使用者空間生態系 (Day 41-50)

**目標：** 打造 User App 的開發環境，並實作系統殼層。

* **Day 41-44:** 建立 User Space 專用的標準 C 函式庫 (迷你版 libc，封裝底層 Syscalls)。
* **Day 45-47:** 實作鍵盤輸入緩衝區與 TTY (終端機) 介面。
* **Day 48-50:** 實作命令列殼層 (Shell)，讓使用者能自由執行硬碟內的外部指令如 `ls`, `cat`, `echo`。

### 第六階段：圖形介面與視窗系統 (Day 51-60)

**目標：** 脫離純文字模式，進入高解析度的畫布與視窗世界。

* **Day 51-54:** VBE (VESA BIOS Extensions) 啟動與線性幀緩衝 (Framebuffer) 渲染。
* **Day 55-57:** 基礎圖形引擎開發（畫點、線、矩形）與字體解析渲染。
* **Day 58-60:** 實作簡單的視窗合成器 (Compositor) 與滑鼠游標支援，完成 GUI 作業系統雛形。

### 給你的建議與反饋

1. **調試是最大的挑戰：** 在 OS 開發中，你沒有 `gdb` 可以直接連。建議先學會如何使用 QEMU 的偵錯開關（如 `-d int` 查看中斷）以及使用 GDB 遠端連接 QEMU 核心。
2. **不要過度設計：** 在這 60 天內，目標是「理解概念」而非做出「下一個 Linux」。例如：檔案系統選 FAT16 就好，不要挑戰實作實時日誌系統。
3. **關於硬體：** 既然你使用的是 **MacBook Pro M1 Max**，你需要建立一個 x86_64 的 Docker 容器來進行編譯，或者使用虛擬機運行 Linux，因為 macOS 的 Linker 與 Linux 格式 (ELF) 不同。
4. **每天兩小時的分配：**
* **前 30 分鐘：** 閱讀理論（例如：今天要做 IDT，先看 Intel 手冊關於 Interrupts 的章節）。
* **後 90 分鐘：** 實作與 Debug。
