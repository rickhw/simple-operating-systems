# Q3: 可以針對現在 boot.S 記憶體，以及 kernel.c 這兩個初始檔，製作一段簡單動畫呈現記憶體狀況，以及運作過程？

雖然我無法直接在這裡為你算圖生成一個 MP4 影片或 GIF 檔，但我可以為你製作一個**「文字版的分鏡動畫 (Storyboard)」**！

這也是許多資深工程師在腦中 Debug 時的畫面。我們用「影格 (Frame)」的方式，放慢時間，一起觀察 CPU 暫存器（Registers）、堆疊（Stack）和 VGA 影像記憶體在 `boot.S` 交接給 `kernel.c` 時的瞬間變化。

---

### 🎬 核心啟動微觀動畫：From Assembly to C

#### ⬛ Frame 1：GRUB 剛交接控制權 (進入 `_start`)

Bootloader 剛把我們的核心載入到 `1MB (0x100000)` 的位置，並把 CPU 控制權交給了 `_start` 的第一行。此時 C 語言還沒啟動，堆疊也是未知的。

* **CPU 指令指標 (`EIP`)：** `mov esp, stack_top` (準備執行)
* **CPU 堆疊指標 (`ESP`)：** `0x????????` (未知狀態，充滿垃圾值)
* **VGA 記憶體 (`0xB8000`)：** 畫面上還留著 GRUB 的開機選單文字 (例如 `0x074D` 即灰底白字的 'M')。
* **Stack 記憶體：** 16KB 的空間全都是 `0x00` (因為在 `.bss` 區段被初始化為 0)。

---

#### 🟩 Frame 2：建立堆疊 (Stack 準備完畢)

CPU 執行了 `mov esp, stack_top`。這是歷史性的一刻，C 語言的舞台搭好了！

* **CPU 指令指標 (`EIP`)：** `call kernel_main` (準備跳往 C 語言)
* **CPU 堆疊指標 (`ESP`)：** `0x00105000` (假設 `stack_top` 的位址，精準指向 16KB 空間的最高處)
* **VGA 記憶體 (`0xB8000`)：** 不變 (依然是 GRUB 殘影)。
* **Stack 記憶體：** ```text
0x00105000 (stack_top): 0x00000000 <-- ESP 正指著這裡
...
0x00101000 (stack_bottom): 0x00000000
```


```



---

#### 🟨 Frame 3：跳躍進入 C 語言 (`kernel_main`)

執行 `call kernel_main`。CPU 為了等一下能回到組合語言，會把下一行指令 (`cli`) 的位址「推 (Push)」進堆疊裡。

* **CPU 指令指標 (`EIP`)：** `terminal_initialize()` (進入 C 語言的第一行)
* **CPU 堆疊指標 (`ESP`)：** `0x00104FFC` (往下移動了 4 bytes！)
* **VGA 記憶體 (`0xB8000`)：** 不變。
* **Stack 記憶體：**
```text
0x00105000 (stack_top): [存放了 boot.S 裡 cli 指令的返回位址]
0x00104FFC: 0x00000000 <-- ESP 現在指著這裡
...

```



---

#### 🟦 Frame 4：執行清空螢幕 (`terminal_initialize`)

C 語言的雙層 `for` 迴圈開始瘋狂運轉。我們把黑底白字的空白字元 `0x0F20` (`0x0F` 是顏色，`0x20` 是空白 ASCII) 寫滿整個畫面 (80 x 25 = 2000 個字元)。

* **CPU 指令指標 (`EIP`)：** `terminal_writestring(...)` (準備印字)
* **CPU 堆疊指標 (`ESP`)：** 微微往下移動，因為 `for` 迴圈使用了區域變數 `x`, `y`, `index`。
* **VGA 記憶體 (`0xB8000`)：** ```text
0xB8000: 0x0F20 (空白)
0xB8002: 0x0F20 (空白)
... 全部被洗成黑色畫面！
```


```



---

#### 🟥 Frame 5：印出綠色字串 (`terminal_writestring`)

`while` 迴圈啟動，把 "Hello..." 依序寫入 VGA 記憶體的最前端。綠色是 `0x0A`。

* **CPU 指令指標 (`EIP`)：** 函式執行完畢，準備 Return。
* **VGA 記憶體 (`0xB8000`)：** ```text
0xB8000: 0x0A48 (綠色的 'H')
0xB8002: 0x0A65 (綠色的 'e')
0xB8004: 0x0A6C (綠色的 'l')
0xB8006: 0x0A6C (綠色的 'l')
0xB8008: 0x0A6F (綠色的 'o')
... 螢幕上正式浮現我們的綠色字串！
```


```



---

#### ⬛ Frame 6：完成與休眠 (回到 `boot.S`)

C 語言的 `kernel_main` 執行結束，CPU 從堆疊中彈出 (Pop) 先前存入的返回位址，回到了 `boot.S` 的 `.hang` 標籤。

* **CPU 指令指標 (`EIP`)：** `hlt` (進入休眠指令)
* **CPU 堆疊指標 (`ESP`)：** `0x00105000` (回到原點，區域變數的空間被釋放)
* **系統狀態：** 完美停留在印著綠字的畫面上，CPU 進入低耗電休眠，等待未來我們加入「中斷 (Interrupt)」來喚醒它。

---

這就是從 Bootloader 轉交權限，到 C 語言在畫面上印出第一個字元的完整內部運作微觀過程！

你可以把這個過程想像成一次完美的「接力賽」：GRUB 跑完第一棒交給 `boot.S`，`boot.S` 趕緊把跑道（Stack）鋪好，然後把接力棒交給 C 語言衝刺出最後的畫面。
