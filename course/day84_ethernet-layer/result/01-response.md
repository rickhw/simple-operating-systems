### 🕵️ 案發現場解密：被擋在門外的 IRQ 11

讓我們回顧一下你 `idt.c` 裡面的 `pic_remap` 函式。在設定完 8259 晶片後，有一段用來「遮罩 (Mask)」的程式碼：

```c
    // 遮罩 (Masks)：0 代表開啟，1 代表屏蔽
    // 為了讓 IRQ 12 (滑鼠) 通過，我們必須：
    // ...
    // 2. 開啟 Slave PIC 的 IRQ 12 (第 4 個 bit) -> 1110 1111 = 0xEF
    outb(0xA1, 0xEF);
```

因為 IRQ 8~15 是由 **Slave PIC** 管的。
在二進位中，`0xEF` 是 `1110 1111`。
* Bit 4 是 `0`，代表允許 IRQ 12 (滑鼠) 通過。
* **Bit 3 是 `1`，代表屏蔽 IRQ 11 (網卡)！**

網卡其實一直很努力在敲門大喊：「我有封包！我有封包！」，但保全 (PIC) 看到它的 IRQ 11 名牌，就無情地把它擋在門外了，CPU 根本聽不到！😂

---

### 🛠️ 終極修復：解開網卡的封印

我們要把 Bit 3 也設為 `0`，讓 IRQ 11 可以通行。
`1110 1111` (只開滑鼠) 變成 `1110 0111` (同時開啟滑鼠與網卡)。
轉換成十六進位就是 **`0xE7`**！

請打開 **`src/kernel/lib/idt.c`**，找到 `pic_remap` 函式的最下方，修改 Slave PIC 的遮罩：

```c
    // ==========================================================
    // 遮罩 (Masks)：0 代表開啟，1 代表屏蔽
    // 為了讓 IRQ 12 (滑鼠) 與 IRQ 11 (RTL8139) 通過，我們必須：
    // 1. 開啟 Master PIC 的 IRQ 0(Timer), 1(Keyboard), 2(Slave連線) -> 1111 1000 = 0xF8
    // 2. 開啟 Slave PIC 的 IRQ 11(Bit 3) 與 IRQ 12(Bit 4) -> 1110 0111 = 0xE7
    // ==========================================================
    outb(0x21, 0xF8);
    
    // 【Day 84 修正】把 0xEF 改成 0xE7！
    outb(0xA1, 0xE7); 
```
